Dreamer 1.2.js Architecture

Single-file Node.js trading bot runtime: CLI operator entry point, Robinhood API wrapper, live trading engine, regime memory, and forked Dreamer optimizer workers connected by IPC and JSON persistence.

Runtime Topology

  • • mainLoop() initializes the process and runs the trading cycle.
  • • TradingEngine owns live portfolio state and order execution.
  • • LegionManager decides when to cull, spawn, or optimize shadows.
  • • ScientificOptimizer runs in forked simulation workers.

Persistence & Recovery

  • • liveEngineState.json snapshots baselines, holdings, and genome.
  • • configs/asset_regime_memory.json persists regime memory across restarts.
  • • configs/sweep_state_*.json preserves optimizer scan progress per worker.
  • • market_data.jsonl and trade_history.log feed simulations and slippage analysis.

Control & IPC Flow

  • • Environment secrets are used only for Robinhood signing and request auth.
  • • Worker feedback travels over process.send() / process.on('message').
  • • Optimizer discoveries hot-swap into the live genome after verification.
  • • Regime memory hydrates the detector before each live cycle begins.
